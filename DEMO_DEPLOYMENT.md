# ShangTech 官网 Demo · 大会现场部署说明

## 概述

本文档面向大会现场演示 ShangTech 官网 + CMS 后台的部署与运维人员，覆盖现场配置、风险应对、演示流程三个核心场景。预期受众：1-2 名现场运维人员，对 Windows 基本操作熟悉。

**演示规模**：约 700 名参会者分批访问，峰值并发预计 20-30 人。

**演示内容**：官网首页全流程 / 申请试用表单 / 产品·案例详情页 / CMS 后台。

## 一、现场配置清单

### 1.1 硬件清单

| 物品 | 数量 | 用途 |
|---|---|---|
| 主演示电脑（Windows 10/11，16GB+） | 1 | 跑 web/cms/api 三个生产服务 |
| 备用电脑（同上配置） | 1 | 主机故障时切换，提前装好环境并验证通过 |
| 家用 WiFi 路由器（千兆 LAN 口） | 1 | 独立 WiFi，与电脑用网线直连 |

| 路由器电源适配器 | 1 | |
| 演示用大屏 / 投影（可选） | 1 | 主演示电脑投屏，便于自助观看 |
| 充电宝 / 排插 | 1-2 | 现场电源可能远离展位 |

### 1.2 备用机代码同步方式

备用机代码可放到**任意位置**，不要求跟主电脑路径一致。

**方式 A —— 用 Git 拉取（推荐）**：

```bash
cd C:\Users\备用用户名
git clone <仓库地址> shangtech-website
cd shangtech-website
pnpm install
pnpm run build
```

**方式 B —— U 盘拷贝**：
把主电脑整个 `shangtech-website` 文件夹复制到备用机任意位置（如 `D:\demo`）。已包含构建产物，备用机只需确认 PostgreSQL 已安装运行，即可直接 `pnpm start`。

> ⚠️ 如果主备电脑 CPU 架构差异大（如主电脑 Intel、备用机 AMD），`pnpm build` 产生的原生模块可能不兼容，建议在备用机上重新跑一遍 `pnpm run build`。

### 1.3 软件准备（会议前一天完成）

在主演示电脑和备用电脑上都要准备好：

1. **代码同步**：确保 `d:\My\demo\shangtech-website` 代码在两台电脑上一致
2. **依赖安装**：`pnpm install`（如已安装可跳过）
3. **生产构建**：`pnpm run build` — 三个应用都要构建成功
4. **数据库**：PostgreSQL 15 运行在 localhost:5432，库 `shangtech_website`，用户 `postgres / postgres`（与 `.env` 一致）
5. **第一次启动验证**：`pnpm start` 跑起来，浏览器访问 http://localhost:3000 和 http://localhost:3001/login 都正常

### 1.3 路由器配置（会议前一天晚上）

1. 路由器恢复出厂设置，重置 WiFi 名称为 `ShangTech-Demo`，密码设为简单易输入的（如 `shangtech2026`）
2. 关闭路由器的"客户端隔离"功能（不同路由器叫法不同：AP 隔离 / 访客网络隔离），否则手机能连 WiFi 但访问不到你电脑
3. DHCP 段设为 `192.168.50.100-200`，电脑固定 IP 设为 `192.168.50.10`（避免重启后 IP 变化导致用户访问失效）
4. 测试：手机连上 `ShangTech-Demo` WiFi，浏览器访问 `http://192.168.50.10:3000` 能打开官网

### 1.4 Windows 系统配置

**电源设置**（避免演示中睡眠）：
- 设置 → 系统 → 电源 → 屏幕和睡眠 → "接通电源时，使设备进入睡眠状态"设为"从不"
- 屏幕关闭时间设为"从不"或"30 分钟"

**防火墙**（首次外部访问会弹授权）：
- 提前一天用别的设备访问你电脑 IP，触发 Windows Defender 防火墙弹窗
- 弹窗点"允许专用网络"和"允许公用网络"
- 或手动入站规则放通 TCP 3000、3001、4000 三个端口

**WiFi 优先级**：
- 电脑连接 `ShangTech-Demo` 路由器（用网线）
- 断开会场 WiFi，避免 Windows 网络路由错乱
- 命令行 `ipconfig` 确认 IPv4 地址是 `192.168.50.10`

## 四、风险点与应对

### 风险 1：电脑睡眠/锁屏导致服务断开

**预防**：
- 电源设置改为"从不睡眠"（已在配置阶段完成）
- 关闭屏幕保护程序
- 演示期间不要按 Win+L 锁屏

**应急**：
- 如发生，唤醒后等待 10 秒，三个服务会自动恢复
- 浏览器刷新当前标签页

### 风险 2：路由器断电或卡死

**预防**：路由器电源线用胶带固定，避免被人绊倒

**应急**：
- 拔插路由器电源，30 秒后重启完成
- 期间暂停演示，让客户用手机 4G 浏览官网缓存版本（无效，因为客户端 JS 还要请求 API）——此情况下直接切到备用机方案

### 风险 3：服务进程崩溃

**症状**：浏览器访问 http://192.168.50.10:3000 显示连接拒绝或 500

**排查**：
```bash
# 看三个端口是否还在监听
netstat -ano | findstr ":3000 :3001 :4000"
```

**应对**：
- 缺哪个端口就重启哪个，最简单是直接 Ctrl+C 终止 `pnpm start`，重新 `pnpm start`
- 重启需要约 15 秒（生产模式），期间浏览器会显示连接超时

### 风险 4：数据库被乱提交污染

**症状**：CMS 询价管理列表里出现大量测试/垃圾数据

**应对**：
```bash
# 用 psql 连数据库清空询价表
psql -U postgres -d shangtech_website -c "TRUNCATE inquiries RESTART IDENTITY;"
```

或用 pgAdmin / DBeaver 连库执行 `TRUNCATE inquiries;`

**预防**：演示前清理一次表，避免之前的测试数据混入。

### 风险 5：网络拥塞（700 人同时连）

**症状**：用户加载页面很慢或加载失败

**应对**：
- 提前用 autocannon 或 ab 压测：`autocannon -c 50 -d 30 http://192.168.50.10:3000/`
- 如果 50 并发就崩，说明电脑扛不住，分批让用户进入（每批 20 人，3-5 分钟轮换）
- 提示用户：「请用手机连接 ShangTech-Demo WiFi，扫码或访问 192.168.50.10:3000」

### 风险 6：Windows 自动更新弹窗

**预防**：
- 设置 → Windows 更新 → 暂停更新 4 周
- 会议前一天手动检查一次更新并装完

**应急**：
- 如发生，弹窗选"稍后提醒"，不要在演示中重启
- 强制更新无法跳过时，切到备用机

### 风险 7：备用机切换

**触发条件**：
- 主演示电脑死机无法重启
- Windows 蓝屏
- 主机硬件故障

**流程**：
1. 切换路由器网线到备用电脑
2. 备用电脑提前已设好同 IP `192.168.50.10`（否则路由器 DHCP 给新 IP，要重新贴二维码）
3. 备用电脑上 `pnpm start` 启动服务
4. 浏览器标签页刷新即可继续演示

**关键**：备用电脑会议前一天必须用同样的 `pnpm build` 构建过、`pnpm start` 跑通过一次、数据库有相同的初始数据。

## 五、监控要点

### 5.1 简单监控（不装额外工具）

打开任务管理器，演示期间关注：
- **CPU**：保持在 60% 以下为正常；持续 100% 说明有进程卡死
- **内存**：16GB 机器占用 12GB 以下为正常；超过 14GB 可能开始用 swap，响应变慢
- **进程**：确保有 3 个 `node.exe` 进程在跑（web / cms / api）

### 5.2 实时检查服务可用性

打开一个 PowerShell 窗口，每隔几分钟手动跑：

```powershell
curl http://localhost:3000; curl http://localhost:3001; curl http://localhost:4000/api/products
```

每个都返回 200 即正常。

## 附录：常见问题速查

**Q：用户连上 WiFi 但打不开网页？**
A：检查路由器是否关闭了"客户端隔离"；检查电脑防火墙是否放通 3000 端口；`ipconfig` 确认电脑 IP 没变。

**Q：表单提交后页面没反应？**
A：F12 开浏览器控制台看 Network，看 `/api/inquiries` 请求是 200 还是失败。失败多半是 API 服务挂了，重启 `pnpm start`。

**Q：CMS 后台登录提示 401？**
A：密码错。确认是 `admin / admin123`（区分大小写）。如改过密码忘了，连数据库 `UPDATE users SET password = '<bcrypt hash of admin123>' WHERE username = 'admin';` 重置。

**Q：演示中想改一个产品的文案？**
A：CMS 后台「产品管理」找对应产品点「编辑」，改完保存。前端刷新即可看到新内容（生产模式缓存可能要 10 秒生效）。

**Q：电脑突然没声音/弹通知打断演示？**
A：演示前关闭所有通知：设置 → 系统 → 通知 → 关闭"获取来自应用和其他发送者的通知"。专注助手开启"演示"模式。

**Q：700 人同时访问时浏览器卡死？**
A：服务端不会卡（生产模式能扛），是用户手机浏览器渲染慢。建议用户用 Chrome/Safari，关闭其他后台 App。如确实大量用户卡，分批进入展位。
